### 2. 算法

#### 2.1 T0级：几乎必用，绝对核心
- 线性规划 / 整数规划 (LP / IP / MILP)：出现在超过50%的优化和决策类题目中，是国赛的基石。
  - 实战经验：常用于资源分配、生产计划、运输问题、任务指派。建议熟练使用 Lingo 或 MATLAB/Python (PuLP, Cvxpy) 进行求解。
- 遗传算法 (GA)：函数优化、TSP、调度等NP-hard问题的“万能钥匙”，尤其在问题复杂、约束非线性时表现出色。
- 模拟退火 (SA)：常与遗传算法互补，用于改良圈、TSP、布局优化等。
- 图论算法 (Dijkstra/Floyd, Kruskal/Prim, 网络流)：解决路径规划、最小生成树、网络容量、竞赛排名等问题。是国赛的常青树。
- 层次分析法 (AHP)：评价类问题首选，用于确定权重，操作简单，原理易懂，历届论文中出现率极高。
- 插值与拟合：所有处理连续数据的问题都可能用到，如预测、图像处理、物理量估计。是数据预处理和模型建立的基础。
#### 2.2 T1级：高频出现，必须熟悉
- TOPSIS：综合评价的经典方法，常与熵权法结合，用于客观赋权并排序。
- 熵权法：客观确定权值的首选，与AHP、TOPSIS形成黄金组合。
- 回归分析 (线性/多元/多项式)：最基础的预测方法，是复杂模型的baseline，相对容易建立且解释性强。
- 时间序列 (ARIMA)：单变量预测的标准模型，在销量、温度、股价等预测问题中高频出现。
- 聚类分析 (K-Means, 层次)：处理分类、分割、识别、群组发现等问题的基础方法。
- 微分方程 (ODE)：物理、化学、生物（如种群、传染病）等机理建模的核心工具。SIR/SEIR模型是近年来疫情相关问题的标准解法。
- 支持向量机 (SVM/SVR)：小样本、非线性分类和回归的利器，在数据维度不高时效果很好。
#### 2.3 T2级：特定场景出现，选择性掌握
- 动态规划 (DP)：解决多阶段决策问题（如背包、资源分配、设备更新），最优子结构清晰时效果最佳。
- 粒子群算法 (PSO)：相比遗传算法，实现更简单，参数更少，快速寻优能力强。
- 蚁群算法 (ACO)：专门用于TSP、路径规划问题，路径搜索能力强。
 -蒙特卡洛模拟：处理不确定性问题、随机过程（如风险分析、排队论）的基石。
- 朴素贝叶斯 / KNN：基础分类算法，可作为机器学习分类任务的baseline。
#### 2.4 T3级：提升档次，展现深度
- 多目标优化 (NSGA-II, MOPSO)：用于有多个冲突目标的优化问题（成本vs效率，风险vs收益），是冲击国一的利器之一。
- 集成学习 (随机森林, XGBoost, LightGBM)：在大数据量、高精度预测任务中表现出色，是冲击国一的利器之二。
- 神经网络 (BP, LSTM, CNN)：处理复杂非线性映射、时间序列、图像识别等问题，需要较大数据量，是冲击国一的利器之三。
- 组合算法 (GA+SA, ACO+2-opt)：将两种算法混合以取长补短，是论文的创新点来源。

### 3. 标准数学建模思路

1. 题意重述 + 问题拆解：用自己的话复述，将一个大问题分解为几个逻辑清晰的小问题。
2. 符号与变量定义：建立符号表，明确输入、输出、决策变量。
3. 建模假设：列出必要且合理的假设，说明为何简化。假设是模型成立的基础，务必言简意赅。
4. 数据清洗与特征构造：处理缺失值、异常值，根据问题背景构造有意义的特征。
5. baseline 模型建立：先用最简单的经典模型（如线性回归、AHP）得到一个可行解作为基准。
6. 改良模型设计：针对baseline的不足（如精度不够、考虑因素单一），引入更复杂的模型或进行改进（如引入熵权法、加入非线性项）。
7. 实验对比与可视化：将baseline与改良模型的结果进行定量对比，并用清晰的图表展示。
8. 稳健性与敏感性分析：改变参数或假设，检验结果是否依然可靠。这是论文质量的保证。
9. 结论与建议：给出具体、量化的结论，并基于结果提出有实际操作意义的建议。
10. 局限性与改进方向：诚实地指出模型的不足之处，并展望未来可以如何改进。

### 4. 论文写作系统

#### 4.1 章节结构

1. 问题重述
2. 模型假设
3. 符号说明
4. 模型建立
5. 求解方法
6. 结果分析
7. 模型评价 (含灵敏度/稳定性分析)
8. 结论与建议

#### 4.2 算法段落固定模板

1. 原法简介 (3-5行)：简单说明该算法的基本原理和适用场景。
2. 原方法不足 (结合本题语境)：分析标准算法在解决本题时可能存在的问题。
3. 改进策略 (1-3点)：列出你的具体改进措施（如：引入动态权值、加入惩罚函数、改进交叉算子）。
4. 实验验证 (baseline + 改进)：通过具体数据或图表证明你的改进有效。
5. 结论：明确指出改进是否显著提升了模型性能（精度、效率、鲁棒性等）。

#### 4.3 图表最低要求

- 每个关键模型至少 1 张核心图：流程图、模型结构图、对比图。
- 每个实验至少 1 张对比表：baseline结果、本模型结果、相对性能。
- 图中必须有坐标、单位、图例。
- 表必须为三线表：顶线、表头线、底线。

### 5. 代码工程规范

- 函数化与入口: 每个脚本必须函数化，保留 if __name__ == "__main__": 入口。
- 语义化命名: 变量名语义化，不使用无意义单字符变量（循环索引除外）。
- 模块化: 每个算法模块至少包含：input()、solve()、output()、visualize()。
- 版本控制：每次改良必须保留baseline对照文件，便于对比。
- 注释规范: 关键步骤、参数、公式出处必须写注释。不要写“注释了每一行代码”，而是解释“为什么这样做”。

### 6. 数据处理与实验规范

- 缺失值处理：明确记录选择和理由（删除/均值插值/模型填补）。
- 异常值处理：明确记录阈值和理由（IQR/3σ/模型法）。
- 数据标准化：明确记录公式和理由（MinMax / Z-score），特别是对距离类算法（SVM, KNN, KMeans）至关重要。
- 随机数种子：所有涉及随机性的实验（如GA, SA, 蒙特卡洛）必须设置随机数种子，确保结果可复现。
